## A. 경영진 요약

### A1. 제품 비전

> **"냉장고 사진 한 장으로 오늘 저녁 완성."**

AI가 식재료를 자동으로 인식·관리하고, 보유 재료 기반으로 만들 수 있는 레시피를 AI가 직접 생성해 추천하는 웹 서비스.

### A2. 문제 정의

- 냉장고 속 재료를 일일이 파악하지 못해 음식이 낭비됨
- 유통기한이 지난 재료를 모르고 방치하는 경우가 빈번함
- "오늘 뭐 먹지?" 결정 피로가 반복됨
- 레시피 앱의 재료 입력이 번거로워 지속 사용이 어려움

### A3. 타깃 사용자

- **1인 가구·자취생**: 식재료 관리가 익숙하지 않은 20~30대
- **맞벌이 부부**: 퇴근 후 빠른 식사 결정이 필요한 계층
- **건강·식단 관리 관심층**: 보유 재료 기반 식단을 계획하려는 사용자

### A4. 핵심 가치 제안

- 사진 최대 2장으로 자동 식재료 인식 → 수동 입력 불필요
- AI(GPT-4o)가 냉장고 재료에 맞는 레시피를 직접 생성 — 2단계(후보 제안 → 상세 레시피)
- 월정액 구독 기반의 단순하고 명확한 요금 체계
- Google 소셜 로그인으로 별도 회원가입 불필요

## B. 목표 및 비목표

### B1. 비즈니스 목표

1. 월 구독 전환율 10% 이상 (무료 → 유료) 달성
2. 월 활성 사용자(MAU) 중 60% 이상이 주 1회 이상 사진 분석 기능 사용
3. 초기 6개월 내 유료 구독자 500명 확보
4. 외부 AI API 비용을 매출의 30% 이하로 유지

### B2. 사용자 목표

- 사진 한 장(또는 두 장)으로 식재료를 손쉽게 등록·관리
- 보유 재료 기반으로 오늘 만들 수 있는 레시피 추천 수령
- 식재료 낭비 감소 및 유통기한 관리

### B3. MVP 목표 (달성 완료)

- 핵심 흐름 완성: Google 로그인 → 사진 업로드 → AI 분석 → 냉장고 관리 → 레시피 추천
- 무료 플랜(월 5회 분석 / 10회 레시피) + 유료 플랜(월 30회 분석 / 30회 레시피)
- Google OAuth 기반 안전한 인증
- Vercel (프론트) + Railway (백엔드) 배포 파이프라인 운영 중

### B4. 명시적 비목표 (Phase 1 제외)

| 항목 | 이유 |
|------|------|
| 가족/팀 계정 공유 | 복잡도 증가, MVP 후 검토 |
| 모바일 앱 (iOS/Android) | 웹 우선 전략 |
| 관리자 대시보드 (외부 공개) | 내부 AdminPanel 컴포넌트로 운영 중 |
| **영양 정보 분석** | ~~Phase 2 예정~~ → **GPT-4o 생성 레시피 기준 표시 구현 완료** (RECIPE-08 DONE) |
| 소셜 기능(팔로우, 공유 등) | Phase 2 이후 검토 |
| 다국어 지원 | 한국어 우선, 추후 확장 |
| 유통기한 자동 OCR 인식 | 정확도 미검증, Phase 2 예정 |
| 이메일/비밀번호 로그인 | Google OAuth만 지원, 추후 추가 고려 |

## D. 사용자 스토리

### D1. 회원가입 / 로그인

- 사용자로서, Google 계정으로 빠르게 로그인하여 별도 비밀번호 없이 서비스를 이용하고 싶다.
- 신규 사용자로서, 첫 로그인 시 무료 플랜 한도(월 5회 분석 / 10회 레시피)를 명확히 안내받고 싶다.
- 기존 사용자로서, 재로그인 시 이전 냉장고 상태와 설정이 그대로 유지되길 원한다.
- 사용자로서, 로그인 전에는 서비스 소개 랜딩 페이지만 보이고, 로그인 후 대시보드로 자동 이동하길 원한다.

### D2. 사진 업로드 및 AI 분석

- 사용자로서, 냉장고 사진을 최대 2장 업로드하면 AI가 자동으로 재료를 인식해 주길 원한다.
- 사용자로서, AI 분석 중 로딩 상태와 예상 소요 시간을 시각적으로 확인하고 싶다.
- 사용자로서, AI가 인식한 재료 목록을 수정(추가/삭제/수량·단위 변경)한 후 냉장고에 반영하고 싶다.
- 사용자로서, 서버 오류나 AI 실패 시 사용 횟수가 차감되지 않음을 신뢰하고 싶다.

### D3. 무료 한도 및 구독 업그레이드

- 무료 사용자로서, 잔여 분석 횟수와 레시피 횟수를 항상 대시보드에서 확인하고 싶다.
- 무료 사용자로서, 한도 소진 시 명확한 업그레이드 안내를 받고 싶다.
- 유료 전환 희망 사용자로서, 결제 페이지로 자연스럽게 유도되어 빠르게 구독을 시작하고 싶다.
- 유료 사용자로서, 구독 취소·갱신·결제 실패 상태를 투명하게 확인하고 싶다.
- 신규 사용자로서, 프리미엄 플랜 첫 구매 시 30일 무료 체험 혜택을 구독 관리 페이지에서 명확하게 안내받고 싶다.

### D4. 식재료 관리

- 사용자로서, 냉장고에 등록된 재료 목록을 카테고리별로 구분하여 조회할 수 있길 원한다.
- 사용자로서, AI 분석 없이 수동으로 재료를 추가할 수 있길 원한다.
- 사용자로서, 재료별 수량·단위·유통기한(선택)을 직접 입력할 수 있길 원한다.
- 사용자로서, 재료의 보관 방법(냉장/냉동/실온)과 부패 주의 여부가 표시되길 원한다.
- 사용자로서, 여러 재료를 동시에 선택하여 한 번에 삭제할 수 있길 원한다 — 유통기한 만료 재료를 한꺼번에 정리하고 싶음.

### D5. 레시피 추천 (AI 생성)

- 사용자로서, 음식 종류(밥류/면류/빵류 등)와 맛(매운/단/짠/다이어트)을 선택하여 맞춤 레시피를 받고 싶다.
- 사용자로서, AI가 제안한 3개 후보 요리 중 원하는 것을 선택하면 상세 레시피를 받고 싶다.
- 사용자로서, 레시피를 북마크하여 나중에 다시 확인하고 싶다.
- 사용자로서, 레시피 상세 페이지에서 칼로리·단백질·탄수화물·지방 정보를 확인하고 싶다 — 식단 관리에 활용하고 싶음.
- 다이어트 관리 사용자로서, AI 레시피 생성 시 "다이어트" 맛 옵션을 선택하여 저칼로리·고단백 위주의 레시피를 추천받고 싶다.

### F2. 쿼터 적용 로직

1. 사용자가 분석 또는 레시피 생성 요청
2. 백엔드: 현재 월 해당 feature('analysis' 또는 'recipe') 사용량 조회 (FOR UPDATE 잠금)

4. AI API 호출 및 처리
5. **성공 응답 수신 후에만** usage_count + 1 (트랜잭션 보장)
6. 실패/타임아웃/서버 오류: 횟수 차감 없음
7. 관리자 계정 (limit_count = 99999): 차감 없이 usage_count만 증가

### F3. 업그레이드 트리거 UX

- 분석/레시피 한도 소진 직전 (잔여 1회): 소프트 알림 링크 표시
- 한도 소진 후 시도: 업그레이드 안내 메시지 + `/subscription` 링크
- 대시보드 상단: 분석 잔여 횟수 항상 가시화 (프로그레스 바)
- 구독 후 즉시 유료 혜택 반영 (Polar.sh 웹훅 처리 완료 기준)
- **구독 관리 페이지**: 비프리미엄 사용자에게 "첫 구매 30일 무료 체험" 프로모션 카드 상시 표시 (업그레이드 버튼 위)

### F5. 첫 구매 무료 체험 정책

| 항목 | 내용 |
|------|------|
| 대상 | 프리미엄 첫 구독 사용자 |
| 혜택 | 30일 무료 체험 (Polar.sh 트라이얼 기간 설정) |
| 취소 정책 | 체험 기간 내 취소 시 요금 없음 |
| UI 표시 위치 | `/subscription` 페이지 비프리미엄 사용자 전용 프로모 카드 |
| 카드 스타일 | Deep Night bg + Prigio Green border + 🎁 아이콘 + Mint 텍스트 |

### F4. 결제 생애주기

- **신규 구독**: Polar.sh 체크아웃 → 결제 완료 웹훅 → DB 구독 상태 `active`
- **갱신**: 자동 갱신 웹훅 → DB 갱신일 업데이트
- **취소**: 취소 웹훅 → 현재 기간 종료 시까지 프리미엄 유지 → `canceled`
- **결제 실패**: `past_due` 상태 → 유예 → 자동 다운그레이드
- **강제 해지**: `revoked` → 즉시 free 전환

> ※ DB에 `polar_subscription_id`가 없는 경우(관리자 수동 부여 등)에도 구독 취소 가능 (DB만 변경)

### G2. 핵심 플로우 — 사진 분석

1. `/analyze`에서 이미지 파일 선택 (최대 2장, 파일 클릭 또는 추가 버튼)
2. 클라이언트: 파일 형식(JPEG/PNG/WEBP) 및 크기(10MB) 사전 검증
3. "분석하기 (N장 · 1회 차감)" 버튼 클릭
4. 백엔드 API 호출 (`multipart/form-data`, 최대 2파일)
5. 백엔드: 쿼터 확인 → GPT-4o Vision 병렬 호출 → 결과 병합 (name 기준 중복 제거)
6. 프론트엔드: 신뢰도 배지 포함 재료 목록 표시 (인라인 편집 가능)
7. 수동 재료 추가 가능 (이름/수량/단위 직접 입력)
8. "냉장고에 반영" 클릭 → 일괄 저장 → `/fridge`로 이동

### G3. 핵심 플로우 — 레시피 생성 (2단계)

1. `/recipes`에서 음식 종류 선택 (밥류/면류/빵류/기타 복수 선택 가능)
2. 맛 선택 (매운음식/단음식/짠음식 복수 선택 가능)
3. "레시피 추천받기" 클릭 → GPT-4o가 현재 냉장고 재료 + 필터 기반으로 후보 3개 생성
4. 후보 카드 중 원하는 요리 클릭 → GPT-4o가 상세 레시피 생성 (재료/조리법/시간)
5. 레시피 카드 표시 → 북마크 버튼으로 localStorage 저장 가능

### G4. 실패 상태 처리
