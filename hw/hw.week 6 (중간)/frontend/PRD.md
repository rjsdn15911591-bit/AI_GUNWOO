## A. 경영진 요약

### A1. 제품 비전

> **"냉장고 속 재료를 찍으면, 오늘 저녁 뭐 먹을지 고민 끝."**

AI가 식재료를 자동으로 인식·관리하고, 신뢰할 수 있는 실제 레시피를 추천해 주는 웹 서비스.

### A2. 문제 정의

- 냉장고 속 재료를 일일이 파악하지 못해 음식이 낭비됨
- 유통기한이 지난 재료를 모르고 방치하는 경우가 빈번함
- "오늘 뭐 먹지?" 결정 피로가 반복됨
- 레시피 앱의 재료 입력이 번거로워 지속 사용이 어려움
- AI 생성 레시피는 신뢰도·실현 가능성이 불분명해 실사용에 부적합

### A3. 타깃 사용자

- **1인 가구·자취생**: 식재료 관리가 익숙하지 않은 20~30대
- **맞벌이 부부**: 퇴근 후 빠른 식사 결정이 필요한 계층
- **건강·식단 관리 관심층**: 보유 재료 기반 식단을 계획하려는 사용자

### A4. 핵심 가치 제안

- 사진 한 장으로 자동 식재료 인식 → 수동 입력 불필요
- 신뢰할 수 있는 실제 레시피 추천 (AI 생성 레시피 지양)
- 월정액 구독 기반의 단순하고 명확한 요금 체계
- MVP 우선 접근으로 빠른 시장 검증 가능

## B. 목표 및 비목표

### B1. 비즈니스 목표

1. 월 구독 전환율 10% 이상 (무료 → 유료) 달성
2. 월 활성 사용자(MAU) 중 60% 이상이 주 1회 이상 사진 분석 기능 사용
3. 초기 6개월 내 유료 구독자 500명 확보
4. 외부 AI API 비용을 매출의 30% 이하로 유지

### B2. 사용자 목표

- 사진 한 장으로 식재료를 손쉽게 등록·관리
- 보유 재료 기반으로 오늘 만들 수 있는 현실적인 요리 추천 수령
- 식재료 낭비 감소 및 유통기한 관리

### B3. MVP 목표

- 핵심 흐름 완성: 회원가입 → 사진 업로드 → AI 분석 → 냉장고 관리 → 레시피 추천
- 무료 플랜(월 5회 분석) + 유료 플랜(무제한) 과금 구조 정상 동작
- Google OAuth 기반 안전한 인증
- Vercel + FastAPI 배포 파이프라인 안정 운영

### B4. 명시적 비목표 (Phase 1 제외)

| 항목 | 이유 |
|------|------|
| 가족/팀 계정 공유 | 복잡도 증가, MVP 후 검토 |
| 모바일 앱 (iOS/Android) | 웹 우선 전략 |
| AI 완전 생성 레시피 | 신뢰도 문제, 검증된 레시피 우선 |
| 관리자 대시보드 | 운영 초기 불필요 |
| 영양 정보 분석 | 데이터 복잡성, Phase 2 예정 |
| 소셜 기능(팔로우, 공유 등) | Phase 2 이후 검토 |
| 다국어 지원 | 한국어/영어 우선, 추후 확장 |
| 유통기한 자동 OCR 인식 | 정확도 미검증, Phase 2 예정 |

## D. 사용자 스토리

### D1. 회원가입 / 로그인

- 사용자로서, Google 계정으로 빠르게 로그인하여 별도 비밀번호 없이 서비스를 이용하고 싶다.
- 신규 사용자로서, 첫 로그인 시 서비스 소개와 무료 플랜 한도(월 5회)를 명확히 안내받고 싶다.
- 기존 사용자로서, 재로그인 시 이전 냉장고 상태와 설정이 그대로 유지되길 원한다.

### D2. 사진 업로드 및 AI 분석

- 사용자로서, 냉장고/식재료 사진을 업로드하면 AI가 자동으로 재료를 인식해 주길 원한다.
- 사용자로서, AI 분석 중 로딩 상태를 시각적으로 확인하고 싶다.
- 사용자로서, AI가 인식한 재료 목록을 수정(추가/삭제/수량 변경)한 후 냉장고에 반영하고 싶다.
- 사용자로서, 서버 오류나 AI 실패 시 사용 횟수가 차감되지 않음을 신뢰하고 싶다.

### D3. 무료 한도 및 구독 업그레이드

- 무료 사용자로서, 잔여 분석 횟수를 항상 대시보드에서 확인하고 싶다.
- 무료 사용자로서, 5회 한도 소진 시 명확한 업그레이드 안내를 받고 싶다.
- 유료 전환 희망 사용자로서, 결제 페이지로 자연스럽게 유도되어 빠르게 구독을 시작하고 싶다.
- 유료 사용자로서, 구독 취소·갱신·결제 실패 상태를 투명하게 확인하고 싶다.

### D4. 식재료 관리

- 사용자로서, 냉장고에 등록된 재료 목록을 언제든 조회·편집·삭제할 수 있길 원한다.
- 사용자로서, AI 분석 없이 수동으로 재료를 추가할 수 있길 원한다.
- 사용자로서, 재료별 수량·단위·유통기한(선택)을 직접 입력할 수 있길 원한다.

### D5. 레시피 추천

- 사용자로서, 현재 냉장고 재료를 기반으로 실제로 만들 수 있는 레시피 목록을 받고 싶다.
- 사용자로서, 부족한 재료가 몇 가지인지 표시된 레시피 목록을 보고 싶다 ("재료 3개 부족").
- 사용자로서, 추천된 레시피가 AI가 즉흥적으로 만들어낸 것이 아닌 검증된 레시피임을 신뢰하고 싶다.

## F. 수익화 요구사항

### F1. 플랜 정의

| 구분 | 무료 플랜 | 유료 플랜 |
|------|---------|---------|
| 월 AI 분석 횟수 | 5회 | 무제한 |
| 레시피 추천 | 동일 | 동일 |
| 식재료 관리 | 동일 | 동일 |
| 정확도/품질 차이 | 없음 | 없음 |
| 결제 방식 | - | Polar.sh 월정액 |

> ※ 무료/유료 간 기능 품질 차이 없음. 유일한 차이는 AI 분석 사용 횟수.

### F2. 쿼터 적용 로직

1. 사용자가 사진 분석 요청
2. 백엔드: 현재 월 사용량 조회 → 한도 초과 여부 확인
3. 한도 초과 시: 402 Payment Required 반환, 업그레이드 유도 UI 표시
4. AI API 호출 및 분석 처리
5. 성공 응답 수신 후에만 `usage_count + 1` (트랜잭션 보장)
6. 실패/타임아웃/서버 오류: 횟수 차감 없음

### F3. 업그레이드 트리거 UX

- 분석 횟수 소진 직전 (잔여 1회): 소프트 알림 배너 표시
- 횟수 소진 후 분석 시도: 업그레이드 모달 팝업
- 대시보드 상단: 잔여 횟수 항상 가시화
- 구독 후 즉시 유료 혜택 반영 (웹훅 처리 완료 기준)

### F4. 결제 생애주기

- **신규 구독**: Polar.sh 체크아웃 → 결제 완료 웹훅 → DB 구독 상태 `activated`
- **갱신**: 자동 갱신 웹훅 → DB 갱신일 업데이트
- **취소**: 취소 웹훅 → 현재 기간 종료 시까지 유료 유지 → `canceled`
- **결제 실패**: `past_due` 상태 → 3일 유예 → 자동 다운그레이드
- **환불**: Polar.sh 정책 적용, 서비스 측 별도 처리 불필요

### G2. 핵심 플로우: 사진 분석

1. 사용자가 `/analyze` 에서 이미지 선택 (드래그앤드롭 또는 파일 선택)
2. 클라이언트: 파일 형식/크기 사전 검증
3. 백엔드 API 호출 (`multipart/form-data`)
4. 백엔드: 쿼터 확인 → AI API 호출 → 결과 정규화
5. 프론트엔드: 인식 재료 목록 표시 (수정 가능)
6. 사용자 확인 → "냉장고에 반영" 클릭 → 재료 일괄 저장
7. 성공 토스트 + 잔여 횟수 업데이트

### G3. 실패 상태 처리

| 실패 시나리오 | 사용자에게 표시 | 횟수 차감 |
|-------------|--------------|---------|
| 이미지 업로드 실패 | 오류 메시지 + 재시도 버튼 | 없음 |
| AI API 타임아웃 | "잠시 후 다시 시도해주세요" 안내 | 없음 |
| AI 결과 파싱 오류 | "분석 실패" + 재시도 유도 | 없음 |
| 쿼터 초과 | 업그레이드 모달 | 없음 |
| 서버 오류 (5xx) | 오류 안내 + 재시도 | 없음 |
| 네트워크 오류 | 오프라인 감지 배너 | 없음 |

### G4. 외부 의존성 장애 대응

- **Google OAuth 장애**: 로그인 불가 안내 + 이미 로그인된 세션은 정상 유지
- **Polar.sh 장애**: 결제 페이지 오류 안내, 기존 구독자 혜택은 DB 기준으로 유지
- **외부 AI API 장애**: 분석 기능 일시 중단 안내, 횟수 미차감, 수동 재료 입력 안내
