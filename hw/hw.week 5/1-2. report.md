## Part 1: 스크립트 실행 결과 분석

### 1-1. 정규화 기법 비교 (`01_regularization.py`)

**실행 결과:** `outputs/01_regularization_plot.png`

MNIST 데이터셋 1,000개 샘플로 4가지 정규화 구성을 10 에포크 학습 후 검증 손실을 비교하였다.

| 구성 | 모델 구조 | 특징 |
|------|-----------|------|
| None | Dense(128) × 2 + Dense(10) | 정규화 없음 — 기준선 |
| L2 | + kernel_regularizer=L2(0.01) | 가중치 크기에 패널티 부여 |
| Dropout | + Dropout(0.5) | 학습 중 50% 뉴런 무작위 비활성화 |
| BatchNorm | + BatchNormalization() | 미니배치 단위 정규화로 학습 안정화 |

**관찰:** 정규화를 적용한 모델(L2, Dropout, BatchNorm)은 None 기준선 대비 검증 손실이 안정적으로 수렴하거나 낮은 수준을 유지하였다. Dropout은 초기 손실이 다소 높으나 에포크가 지남에 따라 과적합 억제 효과가 나타났다. BatchNorm은 수렴 속도가 가장 빠른 경향을 보였다.

### 1-2. 과적합 vs 과소적합 (`02_overfitting_underfitting.py`)

**실행 결과:** `outputs/02_overfitting_underfitting.png`

sin(x) + 가우시안 노이즈 데이터(훈련 20개, 테스트 100개)에 3가지 복잡도 모델을 200 에포크 학습하였다.

| 모델 | 구조 | 현상 |
|------|------|------|
| underfit | Dense(1) | 단순 선형 — 복잡한 패턴 학습 불가 (과소적합) |
| balanced | Dense(16) × 2 | 적절한 복잡도 — 노이즈 제거 후 sin 곡선 근사 |
| overfit | Dense(128) × 3 | 훈련 데이터의 노이즈까지 학습 (과적합) |

**관찰:** underfit 모델은 sin 곡선의 비선형성을 전혀 포착하지 못하였다. overfit 모델은 훈련 데이터에는 완벽히 맞지만 검증 손실이 balanced보다 높게 나타났다. balanced 모델이 일반화 성능이 가장 우수하였다.

### 1-3. 데이터 증강 (`03_data_augmentation.py`)

**실행 결과:** `outputs/03_augmentation_examples.png`

MNIST 첫 번째 이미지(숫자 '5')를 100×100으로 리사이즈 후 3가지 증강 기법을 조합하여 9장의 다양한 변형 이미지를 생성하였다.

| 기법 | 파라미터 | 효과 |
|------|----------|------|
| RandomFlip | horizontal_and_vertical | 수평/수직 뒤집기 |
| RandomRotation | 0.2 (±72°) | 임의 회전 |
| RandomZoom | 0.2 (±20%) | 임의 확대/축소 |

**관찰:** 동일한 원본 이미지에 확률적 변환을 적용하여 훈련 데이터 다양성을 인위적으로 증가시킬 수 있다. 이는 소규모 데이터셋에서 모델의 일반화 능력 향상에 기여한다.

### 1-4. 전이 학습 (`04_transfer_learning.py`)

**실행 결과:** `outputs/04_transfer_learning_summary.txt`

ImageNet으로 사전학습된 MobileNetV2를 기반 모델로 사용하고, GlobalAveragePooling2D와 Dense(1) 헤드를 추가하였다.

```
Model: "sequential"
┌──────────────────────────────┬─────────────────┬───────────┐
│ Layer                        │ Output Shape    │  Param #  │
├──────────────────────────────┼─────────────────┼───────────┤
│ mobilenetv2_1.00_160         │ (None,5,5,1280) │ 2,257,984 │
│ global_average_pooling2d     │ (None, 1280)    │         0 │
│ dense                        │ (None, 1)       │     1,281 │
└──────────────────────────────┴─────────────────┴───────────┘
 Total params:       2,259,265 (8.62 MB)
 Trainable params:       1,281 (5.00 KB)  → 0.06%
 Non-trainable params: 2,257,984 (8.61 MB) → 99.94%
```

**관찰:** 전체 파라미터의 99.94%가 동결(non-trainable)되어 있으므로 학습은 Dense 헤드의 1,281개 파라미터만 대상으로 한다. 이를 통해 소규모 데이터셋과 제한된 연산 자원으로도 고성능 분류기를 구축할 수 있다.

### 1-5. MNIST CNN (`05_mnist_cnn.py`)

**실행 결과:** `outputs/05_mnist_cnn_result.png`

전체 MNIST 데이터(훈련 60,000장, 테스트 10,000장)로 CNN 모델을 3 에포크 학습하였다.

**모델 구조:**
```
Conv2D(32, 3×3, relu) → MaxPool(2×2)
Conv2D(64, 3×3, relu) → MaxPool(2×2)
Conv2D(64, 3×3, relu) → Flatten
Dense(64, relu) → Dense(10, softmax)
```

**결과:**

| 에포크 | Train Acc | Val Acc |
|--------|-----------|---------|
| 1 | ~0.97 | ~0.98 |
| 2 | ~0.98 | ~0.99 |
| 3 | ~0.99 | ~0.99 |

**관찰:** 3 에포크만으로 99% 수준의 검증 정확도를 달성하였다. Fully Connected 네트워크 대비 Conv2D 계층이 공간적 특징(엣지, 곡선)을 효과적으로 추출하므로 높은 성능을 보인다.
